musicd protocol changelog
=========================

Version 1
=========
Initial release.


Version 2
=========

* Additions to syntax:
  * List property:
    key=value1,value2

* Changes to protocol:
  * musicd
    + codecs list
      Defines what codecs the server is able to transcode to on client's
      request. Transcoding support is optional, absence of this field means it
      is not supported.
  * open
    + Request: codec string
      Codec requested for server-side transcoding. Must be one reported by
      server when connecting. Absence of this field means no transcoding is
      requested.
    + Request: bitrate integer
      Bitrate requested for server-side transcoding. Server may or may not
      honor this.
  * track
    + path string
